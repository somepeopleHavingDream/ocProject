- 不三端分离的不足
    - 对数据库访问（ORM）的压力增大
    - 应用升级互相影响
    - 随着业务的增长，项目结构会越来越复杂
    - 访问量增大，单一应用的瓶颈无法放开处理
- 三端分离的优势
    - 应用升级不互相影响（升级CMS，并不会影响PC站）
    - 对于核心服务可以拆分出来，可以快速响应需求
    - 不同应用的服务器数量可以灵活调整
    - 独立服务或应用可以灵活地拆分、扩展
- 三种部署发布上线
    - 对于本地Tomcat部署，是每个开发都应该具备的技能
    - 新浪SAE的部署，可以将应用发布到公网访问
    - 如果你有自己的服务器，可以通过nginx反向代理域名部署
